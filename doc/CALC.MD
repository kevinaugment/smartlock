# 智能门锁计算器规划文档

> **核心原则**：每个计算器必须解决真实用户决策问题，输出可操作建议，避免为SEO凑数的"薄工具"

---

## 优先级分级说明

- 🔴 **Tier 1** - MVP核心计算器（5个）：影响购买决策，必须优先实现
- 🟡 **Tier 2** - 高价值计算器（5个）：快速提升专业度和用户粘性
- 🟢 **Tier 3** - 补充计算器（5个）：丰富内容矩阵，根据反馈选择性实现

**实施建议**：Phase 1先完成Tier 1，用户反馈良好后再扩展Tier 2和3

## 🔴 Tier 1: MVP核心计算器

### 1. 门锁兼容性检查器
**URL**: `/tools/door-lock-compatibility-checker`  
**优先级**: ⭐⭐⭐⭐⭐ (最高价值)  
**真实痛点**: 用户购买智能锁后发现门不兼容，退货率高达30%

#### 输入参数
- 门类型：标准木门/防盗门/玻璃门/铁门
- 锁孔类型：圆形deadbolt/方形mortise/无孔
- 门厚度（mm）
- Backset距离：60mm/70mm/其他
- 区域标准：北美(ANSI)/欧洲(DIN)/中国(GB)

#### 输出结果
- 兼容性评级：🟢直接兼容 / 🟡需适配器 / 🔴不推荐
- 推荐锁型号列表（3-5款，按兼容度排序）
- 所需配件清单
- 安装难度：⭐简单DIY / ⭐⭐需工具 / ⭐⭐⭐专业安装
- 常见问题提醒

#### 技术实现要点
- **数据库**：50-100款主流锁的规格参数（门厚度范围、backset、锁孔类型）
- **匹配算法**：评分制（门厚40分+backset30分+锁孔30分）
- **页面内容**：2000字+门规格测量指南+常见错误案例

### 2. 总拥有成本(TCO)计算器
**URL**: `/tools/smart-lock-tco-calculator`  
**优先级**: ⭐⭐⭐⭐⭐  
**真实痛点**: 用户只看到"$199"价格，不知5年总成本（电池+订阅可能再花$500）

#### 输入参数
- 锁单价、门数量、安装方式（DIY/$100专业）
- 协议类型：Wi-Fi/Zigbee/Z-Wave/Thread
- 需要Hub：是($100)/否
- 日均开门次数、订阅费($/月/门)
- 电池价格、使用年限(3/5/10年)
- 对比基准：机械锁/其他智能锁

#### 输出结果
- **成本明细表**：初期投入（硬件+安装+Hub）+ 年度运营（电池+订阅）
- **5年总成本** & 单门单天成本
- **成本占比饼图**：识别最大成本驱动因素
- **敏感性分析**：电池寿命-20%、订阅费+$1对TCO的影响
- **优化建议**："电池占35%，建议选低功耗协议可节省$XXX"
- **对比图表**：年度成本曲线（传统vs智能）

#### 核心计算逻辑
```javascript
function calculateTCO(inputs) {
  // 1. 初期成本
  const initialCost = inputs.lockPrice * inputs.doorCount + 
    (inputs.professionalInstall ? inputs.doorCount * 100 : 0) +
    (inputs.needsHub ? 100 : 0);
  
  // 2. 电池寿命（基于协议功耗模型）
  const batteryLifeDays = {
    wifi: 90, zigbee: 365, zwave: 365, thread: 300
  }[inputs.protocol] * adjustForUsage(inputs.dailyUsage);
  
  const annualBatteryCost = (365 / batteryLifeDays) * 
    inputs.batteryPrice * inputs.doorCount;
  
  // 3. 总成本
  const totalCost = initialCost + 
    (annualBatteryCost + inputs.subscription * 12 * inputs.doorCount) * inputs.years;
  
  return { initialCost, annualBatteryCost, totalCost };
}
```

### 3. 电池寿命&功耗对比器
**URL**: `/tools/battery-life-power-comparison`  
**优先级**: ⭐⭐⭐⭐⭐  
**真实痛点**: "Wi-Fi锁 vs Zigbee锁"，最关心多久换一次电池

#### 输入参数
- 对比协议（多选）：Wi-Fi/Zigbee/Z-Wave/Thread/蓝牙
- 日均开门次数（滑块5-50次）
- 额外功能：自动上锁/远程查询/Always-on Wi-Fi
- 环境温度：室温/寒冷/极寒
- 电池类型：AA碱性/AA锂电池/CR123A + 价格

#### 输出结果
- **对比表格**：
  | 协议 | 预计寿命 | 年更换次数 | 年度成本 | 推荐度 |
  |------|---------|-----------|---------|--------|
  | Zigbee | 12月 | 1次 | $4 | ⭐⭐⭐⭐⭐ |
  | Wi-Fi | 3月 | 4次 | $16 | ⭐⭐ |
  
- **柱状图**：各协议年度电池成本可视化
- **时间轴**：显示何时需要更换
- **最优推荐**："基于日均20次开门，推荐Zigbee，5年可节省$60"
- **情景分析**：增加"频繁远程查询"后寿命变化

### 4. 协议选择场景匹配器
**URL**: `/tools/protocol-selection-wizard`  
**优先级**: ⭐⭐⭐⭐  
**真实痛点**: "Wi-Fi/Zigbee/Z-Wave/Matter...我该选哪个？"技术小白困惑

#### 输入参数（场景导向，非技术术语）
- 建筑类型：独栋/联排/公寓/写字楼
- 门数量：1-3 / 4-10 / 11-30 / 30+
- Wi-Fi覆盖：所有门稳定 / 部分弱信号 / 无Wi-Fi
- 是否愿意部署Hub：是/否/不确定
- 现有智能家居：HomeKit/Google/Alexa/SmartThings/无
- 技术能力：小白/爱好者/专业IT
- 预算敏感度：低/中/高
- 最优先需求：超长电池/快速响应/离线可用/生态丰富/开放标准/成本低

#### 输出结果
1. **Top 3推荐方案**（每个包含）：
   ```
   ### 方案1: Zigbee 3.0 ⭐⭐⭐⭐⭐
   推荐理由：
   - ✅ 网状网络适合多门（自动扩展）
   - ✅ 电池寿命长（1年+）
   - ✅ 兼容SmartThings，可升级Matter
   
   权衡点：
   - ⚠️ 需要Hub（$50-100）
   - ⚠️ 初期设置略复杂
   
   推荐产品：Yale Assure Lock 2, Schlage Encode Plus
   估算成本：5门5年TCO $1,200
   ```

2. **不推荐方案及原因**：
   > ❌ Wi-Fi锁：虽无需Hub，但5门5年电池成本$400 vs Zigbee的$100

3. **决策对比表**：
   |  | Zigbee | Z-Wave | Wi-Fi | Thread |
   |--|--------|--------|-------|--------|
   | 电池寿命 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐ |
   | 响应速度 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |

4. **实施路线图**：分阶段部署建议 + 未来升级路径

### 5. 短租ROI计算器
**URL**: `/tools/short-term-rental-roi-calculator`  
**优先级**: ⭐⭐⭐⭐  
**真实痛点**: Airbnb房东犹豫"$300智能锁值得吗？"需要量化收益

#### 输入参数
- 月均订单数、平均入住天数、房源数量
- 当前交钥匙方式：人工/密码箱/物业代收
- 人工成本：交钥匙时间+时薪+交通费
- 风险成本：lockout事故率+平均损失、钥匙丢失率+换锁成本
- 智能锁方案：锁价+安装+订阅费
- 是否对接PMS系统

#### 输出结果
1. **年度收益分析**：
   ```
   💰 年度节省时间: 120小时 (价值$3,000)
   
   成本对比:
   - 当前方案: $4,200/年
     (人工$3,600 + lockout$400 + 钥匙丢失$200)
   - 智能锁: $390/年
     (硬件分摊$60 + 订阅$36 + 电池$12)
   
   年度净收益: $3,810
   ```

2. **回本期**：初期投资$400，月度节省$318，**回本期1.3个月** ⚡

3. **3年ROI**：总投资$400，总收益$11,430，**ROI 2,758%**

4. **情景分析**：
   | 月订单数 | 回本期 | 3年ROI |
   |---------|-------|--------|
   | 10单 | 2.5月 | 1,200% |
   | 20单 | 1.3月 | 2,758% |
   | 30单 | 0.9月 | 4,100% |

5. **隐形收益**：
   - 可接受last-minute订单
   - 提升评分（自助入住+0.2星）
   - 减少深夜交接安全风险
   - 远程协助降低差评

6. **vs其他方案**：密码箱($30) vs 智能锁($300) vs 高端锁($500+)

## 数据准备需求

## 技术架构建议

### 前端
- React Hook Form：表单管理
- Recharts/Chart.js：图表展示
- 状态管理：useState/useReducer（无需Redux）

### 后端
- D1数据库：存储门锁规格、用户计算历史（可选）

- 每个计算器独立URL
- 服务端渲染说明文字
