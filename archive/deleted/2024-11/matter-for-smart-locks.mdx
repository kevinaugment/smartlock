## What is Matter?

### The 30-Second Explanation

**Matter is NOT a new wireless protocol** â€“ it's an **application layer standard** that sits on top of existing protocols (Thread, Wi-Fi, Ethernet).

Think of it this way:
- **Physical layer**  - radio: Thread, Wi-Fi, Ethernet
- **Application layer**  - commands/data: Matter (standardized)

**Goal:** Any Matter-certified lock responds to the same commands regardless of manufacturer, and works with any Matter-compatible ecosystem.

## Matter Door Lock Device Type

### Standardized Features

Matter defines a **Door Lock cluster** with mandatory and optional features:

**Mandatory features:**
- Lock/Unlock commands
- Lock state reporting  - locked/unlocked/jammed
- Door state  - if sensor present
- PIN code management  - add/remove/list users
- Event logging  - lock/unlock events

**Optional features:**
- Auto-relock after timeout
- One-touch locking
- Privacy mode
- Holiday schedules
- Year/day/hour schedules
- RFID credentials
- Fingerprint credentials
- Face recognition credentials
- Alarms  - tamper, forced entry

**This standardization means:** If a lock supports Matter "Door Lock with PIN Code", every platform knows how to add/remove PINs, regardless of manufacturer.

## Security Model

### Matter Security Architecture

Matter implements **defense-in-depth** security:

**1. Device attestation:**
- Each Matter device has factory-provisioned DAC  - Device Attestation Certificate
- Cryptographically proves device is genuine and not tampered
- Issued by manufacturer, verified during commissioning

**2. Secure commissioning:**
- QR code or NFC-based pairing
- Establishes encrypted channel before any data exchange
- Uses PASE  - Password Authenticated Session Establishment or CASE (Certificate Authenticated Session Establishment)

**3. End-to-end encryption:**
- AES-128-CCM for data transmission
- Unique session keys per connection
- Forward secrecy  - past keys don't compromise future sessions

**4. Access control:**
- ACL  - Access Control List per device
- Granular permissions  - who can read status, who can actuate lock
- Multi-admin model  - multiple controllers can manage same device

### Matter Lock-Specific Security

**PIN code security:**
- PINs encrypted in transit
- Stored encrypted on device
- Rate limiting  - lockout after N failed attempts
- PIN complexity requirements  - configurable minimum length

**Physical security:**
- Tamper detection
- Forced entry detection
- Optional requirement for physical proximity during commissioning

### Comparison with Zigbee/Z-Wave

| Security Feature | Zigbee 3.0 | Z-Wave S2 | Matter | Winner |
|-----------------|-----------|-----------|--------|--------|
| **Encryption** | AES-128 | AES-128 | AES-128 | Tie |
| **Key exchange** | Install Code | ECDH | ECDH + DAC | Matter |
| **Device attestation** | No | No | Yes  - DAC | Matter |
| **Multi-admin** | No | No | Yes | Matter |
| **Forward secrecy** | No | No | Yes | Matter |
| **Certification** | Required | Required | Required + stricter | Matter |

**Verdict:** Matter has the most modern security architecture, incorporating lessons learned from Zigbee/Z-Wave.

## Matter vs Zigbee Migration

### Can Zigbee 3.0 Locks Upgrade to Matter?

**Short answer:** Some can, but not all.

**Technical requirements:**
1. **Silicon compatibility:** Zigbee and Thread both use IEEE 802.15.4 physical layer (2.4GHz)
2. **Sufficient memory:** Matter stack requires more RAM/flash than Zigbee
3. **Manufacturer support:** Firmware update must be provided

**Real-world status:**

**Confirmed upgrades:**
- Yale Assure Lock 2  - Zigbee: Matter upgrade announced, rolling out 2024
- Schlage Encode Plus  - Zigbee model: Evaluating Matter support
- Aqara Smart Lock: Matter bridge approach  - hub translates

**Not upgradeable:**
- Older Zigbee 1.2 locks  - different physical layer, insufficient memory
- Budget locks with minimal firmware update infrastructure
- Locks with proprietary Zigbee implementations

**Hybrid approach:**
Some manufacturers offer **bridge mode:** Existing Zigbee lock connects to Matter-compatible hub, which exposes lock to Matter network. Not as seamless as native Matter, but workable.

### Zigbee 3.0 + Matter Dual Mode

**Emerging pattern:** New locks ship with BOTH Zigbee 3.0 and Matter support in same firmware.

**Benefits:**
- Works with legacy Zigbee hubs immediately
- Works with Matter-only ecosystems
- User can choose which to use
- Future-proof

**Examples:**
- Aqara Smart Lock A100 Pro  - 2024 model: Dual Zigbee/Matter
- Yale Assure Lock 3  - upcoming: Native Matter + backward compatible Zigbee

## Common Questions

### Q1: Can I use Matter lock without internet?

**Yes, with Thread-based Matter locks.** Local commands (unlock via keypad, HomeKit on same network) work offline. Remote access requires internet.

**No, with Wi-Fi Matter locks.** Most require internet for cloud services.

### Q2: Is Matter more secure than Zigbee?

**Yes, slightly.** Matter has device attestation and forward secrecy that Zigbee lacks. Both are extremely secure for residential use.

### Q3: Will my Zigbee 3.0 lock become obsolete?

**No, not obsolete.** Zigbee 3.0 will be supported for years. However, new advanced features may be Matter-only.

### Q4: Can I have both Zigbee and Matter locks in same system?

**Yes.** If your hub supports both (SmartThings, Home Assistant), you can mix. They just operate independently.

### Q5: Is Matter ready for production deployments?

**Residential: Yes (with caveats).** Small deployments (1-5 locks) are fine. Expect occasional firmware updates.

**Commercial: Not yet (2024).** Wait for ecosystem maturity. Pilot only.
